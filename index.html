03 — How I think

Three things I keep coming back to.

Operating principles, not buzzwords — each one came out of a project that made me change my mind.

Principle № 01

Stop guessing, start measuring.

You can't optimise what you can't see. Before proposing a solution, I ask what physical event is nobody counting yet? — the answer is usually a free data source hiding in plain sight.

In practice — Turning Lampang's existing CCTV into a zone-level crowd flow map, with zero new hardware spend.
Principle № 02

Value over vanity.

Counting heads is easy. Connecting heads to money is the job. I push dashboards toward metrics a vendor or a mayor can trade on — dwell time, conversion, basket value — not raw footfall that looks impressive on a slide and changes nothing.

In practice — Reframing the brief from "how many people came?" to "what is each zone worth per hour?"
Principle № 03

Insights go where the user already is.

A dashboard nobody opens is a shelved report. I design for the notification layer, not just the web layer. If the officer lives in LINE, the alert lands in LINE — summarised, actionable, with a default recommendation attached.

In practice — Pushing AI-assessed rain & PM2.5 alerts directly into municipal LINE groups, so the message arrives before the crowd does.
